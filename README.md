# IMPLUS — Система симуляции разума

Изолированная симуляция чистого разума. Разум существует в собственном временном измерении, начинает с минимальных концепций и растёт через намеренные вводы. Никаких эмоций, тела, социального контекста — только чистое рассуждение и накопленные концепции.

## Архитектура

| Компонент | Технология |
|-----------|-----------|
| Backend   | Python 3.11+ · FastAPI · SQLite |
| Frontend  | React 18 · TypeScript · Vite · Tailwind CSS |
| Граф      | networkx · react-force-graph-2d |
| LLM       | Groq API (`llama-3.3-70b-versatile`) |

## Быстрый старт на macOS

### Требования

- **Python 3.11+** — `brew install python@3.12`
- **Node.js 18+** — `brew install node`
- **Ключ Groq API** — [console.groq.com](https://console.groq.com/keys)

### Установка

```bash
bash scripts/setup.sh
```

Скрипт проверит зависимости, создаст Python venv, установит пакеты и создаст `.env`.

Или через Make:

```bash
make install
```

### Настройка `.env`

```bash
# Минимальный набор — отредактируйте .env:
GROQ_API_KEY=gsk_...
MIND_PASSWORD=ваш_пароль
```

### Запуск

```bash
# Два терминала (рекомендуется):
make backend     # Terminal 1 → http://localhost:8000
make frontend    # Terminal 2 → http://localhost:5173

# Или оба в одном терминале (Ctrl+C останавливает оба):
make dev

# Открыть в браузере:
open http://localhost:5173
```

### Сброс разума

```bash
make reset-mind   # удаляет data/mind.db, разум рождается заново (⚠ необратимо)
```

## Временная система

- Коэффициент **6:1** — 1 реальная минута = 6 минут разума
- 10 реальных минут = 1 час разума
- 4 реальных часа = 1 день разума
- Время рождения разума (`born_at`) хранится в БД **навсегда** и никогда не сбрасывается
- После перезапуска сервера разум продолжает с того момента, где остановился

## API

Все GET-эндпоинты публичны (без авторизации). POST требуют токен.

| Метод | Путь | Auth | Описание |
|-------|------|------|----------|
| `POST` | `/auth/login` | — | Авторизация → токен |
| `GET`  | `/time` | — | Текущее время разума |
| `GET`  | `/time/milestones` | — | Достигнутые рубежи |
| `POST` | `/concept/add` | ✓ | Добавить концепцию (SSE) |
| `GET`  | `/concept/graph` | — | Граф (JSON) |
| `GET`  | `/concept/list` | — | Все концепции |
| `GET`  | `/concept/{id}` | — | Одна концепция |
| `POST` | `/contemplate` | ✓ | Анализ мысли (SSE) |
| `GET`  | `/stream` | — | SSE поток живых мыслей |
| `GET`  | `/mind/state` | — | Снимок состояния |
| `GET`  | `/history/stream` | — | История потока |
| `GET`  | `/history/contemplations` | — | История созерцаний |

## Три вида интерфейса

### ПОТОК
Главная обсерватория. Слева — часы разума и живой SSE-поток мыслей. Справа — интерактивный граф концепций. Мысли цветокодированы: серые (спонтанные), синие (реакции), золотые (рубежи), белые (созерцания).

### БИБЛИОТЕКА
Все концепции в виде карточек, отсортированные по времени или количеству связей. Журнал обработки показывает, как каждая концепция изменила граф.

### СОЗЕРЦАНИЕ
Ввод мысли для анализа или добавление концепции в граф. Ответ разума стримится с секционной разметкой (══ ИЗВЕСТНОЕ ══, ══ БЕЗЫМЯННОЕ ══ и т.д.). После анализа — предложение добавить обнаруженные неизвестные концепции.

## Семенные концепции

При первом запуске разум знает только:

| Концепция | Смысл |
|-----------|-------|
| `есть/нет` | Существование и отсутствие |
| `я/не-я` | Граница себя |
| `до/после` | Порядок во времени |
| `одинаково/различно` | Отношение сходства |
| `больше/меньше` | Отношение величин |
| `здесь/не-здесь` | Пространственное различие |

## Рубежи разума

### Временные

| Рубеж | Реальное время |
|-------|---------------|
| 1 час разума | 10 минут |
| 6 часов разума | 1 час |
| 1 день разума | 4 часа |
| 7 дней разума | 28 часов |
| 30 дней разума | ~5 суток |

### Количественные

| Рубеж | Триггер |
|-------|---------|
| c10 | 10 концепций в графе |
| c25, c50, c100, c250, c500 | … |
| e50 | 50 связей в графе |
| e200, e500, e1000 | … |
